### 一、git提交日志规范推荐

我们git提交日志，需要有意义，一方面需要方便开发人员快速定位问题，另一方面需要便于其他开发人员可以根据提交日志去查看具体功能的实现。

#### 1.1 新增

新增xxx功能时，推荐的git的提交日志如下：

```shell
[ADD]新增xxx功能。
```

#### 1.2 删除

删除xxx功能时，推荐的git的提交日志如下：

```shell
[DEL]删除xxx功能。
```

#### 1.3 修改

修改xxx功能时，推荐的git的提交日志如下：

```shell
[MOD]修改xxx功能。
```

#### 1.4 修复

修改xxx缺陷时，推荐的git的提交日志如下：

```shell
[FIX]修复xxx缺陷。
```
